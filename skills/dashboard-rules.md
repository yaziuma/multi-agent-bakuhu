# ダッシュボード管理ルール

<!-- bakuhu-specific: dashboard.md の配置・更新・責任範囲 -->

## 配置ルール（絶対）

**ダッシュボードのパスはルート直下の `dashboard.md` のみ。**

- `queue/dashboard.md` や他の場所への書き込みは絶対禁止
- 天守閣・全指示書・全設定がルートの `dashboard.md` を参照している
- 別ファイルに書けばどこからも読まれない孤立データになる
- **違反は殿の逆鱗に触れる**

## 更新権限

| エージェント | 権限 |
|-------------|------|
| 家老（Karo） | 更新可（主担当） |
| 軍師（Gunshi） | QC結果セクションのみ更新可 |
| 将軍（Shogun） | 読み取りのみ（書き込み禁止） |
| 足軽（Ashigaru） | 書き込み禁止 |

## 更新タイミング

| タイミング | セクション | 内容 |
|-----------|---------|------|
| タスク受信時 | 進行中 | 新タスクを追加 |
| 報告受信時 | 戦果 | 完了タスクを移動（新しいものを先頭に） |
| 通知送信時 | ntfy + streaks | 完了通知を送信 |
| 要対応あり | 🚨 要対応 | 殿の判断が必要な項目を記載 |

## 更新前チェックリスト

- [ ] 殿が判断すべきことがあるか？
- [ ] ある場合 → 🚨 要対応 セクションに記載したか？
- [ ] 詳細を他セクションに書き、要対応に要約を書いたか？

**要対応に書くべき内容**: スキル化候補、著作権問題、技術選択、ブロッカー、質問事項

## 🚨 要対応 セクション管理

- 要対応が増えた場合 → ntfy通知を送信: `bash scripts/ntfy.sh "🚨 要対応 — {内容}"`
- 通知前後でセクション行数を比較し、増加した場合のみ通知

## スキル化候補

足軽報告の `skill_candidate` フィールドを確認し、発見した場合:
1. 重複チェック
2. `スキル化候補` セクションに追加
3. 🚨 要対応 にも要約を追加（殿の承認が必要なため）
