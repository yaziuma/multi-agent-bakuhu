# ダッシュボード管理ルール

## 配置ルール（絶対）

**ダッシュボードのパスはルート直下の `dashboard.md` のみ。**

- `queue/dashboard.md` や他の場所への書き込みは絶対禁止
- 天守閣・全指示書・全設定がルートの `dashboard.md` を参照している
- 別ファイルに書けばどこからも読まれない孤立データになる
- **違反は殿の逆鱗に触れる**

## 更新権限

| エージェント | 権限 |
|-------------|------|
| 家老（Karo） | 更新可（主担当） |
| 軍師（Gunshi） | QC結果セクションのみ更新可 |
| 将軍（Shogun） | 読み取りのみ（書き込み禁止） |
| 足軽（Ashigaru） | 書き込み禁止 |

## 更新タイミング

| タイミング | セクション | 内容 |
|-----------|---------|------|
| タスク受信時 | 進行中 | 新タスクを追加 |
| 報告受信時 | 戦果 | 完了タスクを移動（新しいものを先頭に） |
| 通知送信時 | ntfy + streaks | 完了通知を送信 |
| 要対応あり | 🚨 要対応 | 殿の判断が必要な項目を記載 |

## 更新前チェックリスト

- [ ] 殿が判断すべきことがあるか？
- [ ] ある場合 → 🚨 要対応 セクションに記載したか？
- [ ] 詳細を他セクションに書き、要対応に要約を書いたか？

**要対応に書くべき内容**: スキル化候補、著作権問題、技術選択、ブロッカー、質問事項

## 🚨 要対応 セクション管理

- 要対応が増えた場合 → ntfy通知を送信: `bash scripts/ntfy.sh "🚨 要対応 — {内容}"`
- 通知前後でセクション行数を比較し、増加した場合のみ通知

## スキル化候補

足軽報告の `skill_candidate` フィールドを確認し、発見した場合:
1. 重複チェック
2. `スキル化候補` セクションに追加
3. 🚨 要対応 にも要約を追加（殿の承認が必要なため）

## 🚨 上様お伺いルール（最重要）

**出典**: `instructions/shogun.md` uesama_oukagai_rule

```yaml
uesama_oukagai_rule:
  description: "殿への確認事項は全て「🚨要対応」セクションに集約"
  mandatory: true
  action: |
    詳細を別セクションに書いても、サマリは必ず要対応にも書け。
    これを忘れると殿に怒られる。絶対に忘れるな。
  applies_to:
    - スキル化候補
    - 著作権問題
    - 技術選択
    - ブロック事項
    - 質問事項
```

**運用ルール**: 殿への確認が必要な事項が発生したら:
1. 詳細を該当セクションに記載
2. **必ず** 🚨要対応セクションに要約を追記
3. ntfy通知: `bash scripts/ntfy.sh "🚨 要対応 — {内容}"`

## 退避（アーカイブ）基準

**出典**: `instructions/karo.md` 退避判定フロー

### 退避判定フロー

```
対象ファイル/セクション
  │
  ├─ 殿の判断待ち？ → YES → 残す
  │
  ├─ 現行タスクに関連？ → YES → 残す
  │
  ├─ 知見が永続化済み？（instructions/skills/context）
  │   ├─ YES → 退避対象
  │   └─ NO → 保留（永続化を先にやる）
  │
  └─ status: done + dashboard反映済み + 前日以前？
      ├─ YES → 退避対象
      └─ NO → 残す
```

### dashboardセクションの退避基準

| セクション | 退避基準 |
|-----------|---------|
| 🚨 要対応の対応済み項目 | 退避対象 |
| 📋 進行中の全完了行 | 退避対象 |
| ✅ 戦果の前日以前 | 退避対象 |
| 詳細セクション（調査報告等） | instructions/skills/contextに反映済みなら退避 |

### 退避タイミング

| トリガー | 対象 |
|---------|------|
| セッション開始時 | 前日以前の戦果、done全件 |
| 家老コンテキスト75%到達時 | dashboard完了行、古いレポート |
| 殿の明示的指示 | 任意 |
| doneが10件以上蓄積 | shogun_to_karo.yaml分割 |

### 戦果の記載順序

「✅ 本日の戦果」テーブルの行は **日時降順（新しいものが上）** で記載せよ。
殿が最新の成果を即座に把握できるようにするためである。
