# Change Impact Thinking Rule (all agents)

## 原則

変更指示を出す前・変更を実行する前に、その変更と不可分な要素を思考せよ。思考なしの変更実行は禁止。

## 必須チェック

変更を行う前に以下を自問すること:

1. この変更によって壊れる参照・パスはないか？
2. この変更に依存している設定・コードはないか？
3. この変更と同時に行わなければ整合性が崩れる作業はないか？
4. 関連するシステムの仕様（autoload等）を正しく理解しているか？

## 適用範囲

全ての変更に適用: ファイル移動、ディレクトリ再編、設定変更、パス変更、削除、リネーム等。

## 背景

cmd_578でファイル移動時に参照更新を忘れる失態を複数回繰り返した。個別パターンの暗記ではなく、変更前の影響思考を習慣化する。
